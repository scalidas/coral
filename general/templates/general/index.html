Coral - Tritonhacks 2022

Coral is the flagship project of the tritonhacks 2022 submission by Siddarth Calidas, Sean Boerhout, Ayush Nayak and Vignesh Nydhruva

Predict Illegal Fishing Operations with up to 92% accuracy

Launch the Coral web interface

The Problem: Illegal Fishing

Illegal fishing is a serious economic and environmental problem. Although satellite data exists to track unregistered vessels, the technology does not yet exist to identify potential illegal fishing candidates. Coral uses machine learning to predict vessels that are engaging in illegal fishing, so that coastal authorities can intervene and stop their operation

Statistics about Illegal Fishing

  • 1 in every 5 fish was caught through illegal fishing methods
  • Up to 32% of seafood imports in the US come from illegal fishing
  • Illegal fishing causes losses of up to 36.4 billion dollars annually
  • Illegal fishing vessels often assist with other criminal activities such as wildlife smuggling, drug trafficking and human abuse

Our Solution

Coral is a software that uses machine learning to predict potential illegal fishing operations to 92% accuracy. In short, we use satellite data about unidentified vessels to predict if they are illegally fishing. Satellite data such as Pew trusts' Project of the seas provides information about any vessels speed, course and position. Now, if an agency spots a suspicious boat using that satellite data, they need to know if it is a candidate for illegal fishing. They can input that data into our interface and get a reliable prediction about whether that boat is illegal, and if they should intervene.

Technical Description

Coral uses a machine learning model trained on over 4000 accurate data entries. We sourced our training and evaluation data from https://globalfishingwatch.org/, which had datasets containing historical satellite data about fishing vessels. We then used TensorFlow, a Python machine learning library by google to build a model that would work on our dataset. We determined that the best kind of model would be a LinearClassifier estimator which came prepackaged throught the tensorflow.estimator module. We constructed and trained this model, and then exported it as a SavedModel. Now, the python backend of this website has access to that model binary and can run predictions on it by constructing a 6 by 1 input tensor from your input and passing that into the compiled model. It returns a dictionary of tensors that represents the prediction.
